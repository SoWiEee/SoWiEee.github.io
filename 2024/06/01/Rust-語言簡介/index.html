原始資料型態

在 Rust 中宣告數字時，有分成整數跟浮點數，而且分別還細分不同的範圍。不過在宣告時也不是一定要標記型別，Rust 會依整數或浮點數，分別給你 i32 以及 f64。

整數

整數，也就是不帶小數點的數字，根據不同的需求在 Rust 有 8 bit、16 bit、32 bit、64 bit 以及 128 bit 等不同的型別。

浮點數

浮點數其實就是帶有小數點的數字，跟整數一樣，浮點數也有分 bit，但只有 32 bit（f32） 跟 64 bit（f64），而且第 1 個 bit 都是帶正負號的。

字元

在 Rust 裡的字串需要使用雙引號包起來，如果是單引號的則是字元。
字元以 4 bytes 儲存，它可以用來放任何的 Unicode，也就是除了一般的英文數字外，要放中文日文甚至是 Emijo 也行。

布林值

布林值就更單純了，就只有真（true）跟假（false）兩種值。

關於變數的命名慣例較常見的有以下 2 種：

* 駝峰式：例如 HasCat 或是 myStudentScore，透過大小寫做出區隔，樣子就來駱駝背上的駝峰一樣。
* 蛇式：例如 has_cat 或是 my_student_score，中間用底線分開，上上下下的就像蛇一樣。

在其他語言可能沒有太大的差別，但 Rust 的編譯器會警告。
let 宣告的變數要用蛇式，然後 const 宣告的常數應該要全大寫。

陣列（Array）

在 Rust 中，陣列是存放一群相同型別的群組，當陣列在宣告的時候，需要同時宣告型別及數量，但其實也可讓編譯器處理。在做完宣告後，陣列大小就是固定的，所以不能對陣列進行新增或刪除的行為，倘若要修改陣列中的元素，需要加上 mut 的宣告。要注意的是，對於不可修改的陣列，Rust 硬性規定你一定要放好放滿，否則編譯器又會跑出來抱怨了。
如果我們在存取陣列的時候，存取了超過陣列容量的元素，不像 Javascript 默默地得到 undefined，而是會引發 Panic。

當我們要印出陣列中的所有元素時，可以使用 for 迴圈來處理。使用 for...in 可以把陣列裡的元素一個一個拿出來。其中因為 list 本身是陣列，所以需要再透過 .iter() 方法把它轉成 Iterator，才能讓 for ... in 處理。

元組（Tuple）

Tuple 是一個有序的資料結構，與陣列不同的是，tuple 裡面可以放不同的資料型別。同樣，如果不想寫型別宣告，也是能交給編譯器猜。
當然，tuple 也有隨機存取的功能，只是用 .1 這種方式來寫。
